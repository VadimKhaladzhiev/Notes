Уровни изоляции

	DIRTY READS (видны данные посреди чужой транзакции)
	Первый поток добавляет запись в базу
		Второй поток читает запись из базы 
	Первый поток делает откат
	У второго потока запись, которой нет в базе
	
	NONREPEATABLE READS (второй раз читает те же данные, а они уже другие)
	Первый поток читает строку из базы
		Второй поток изменяет строку и делает commit
	Первый поток читает еще раз строку
	Строки из первого и второго чтения не совпадают

	PHANTOM READ (неожиданно появляющиеся данные)
	Первый поток читает данные из базы
		Второй поток добавляет данные и делает коммит
	У первого потока количество данных неверное	
	
	
	UNCOMMITTED
		Защиты нет
	
		Dirty reads
		Nonrepeatable reads
		Phantom read
	
	
	COMMITTED
		Защита от dirty reads (незакоммитинные данные не видны)
		
		Nonrepeatable reads
		Phantom read

	
	NON-REPEATABLE
		Защита от Dirty reads
		Защита от Nonrepeatable reads (при повторном чтении данные возвращаются те же)
		
		Phantom read
		
	
	SERIALIZABLE
		Защита от Dirty reads
		Защита от Nonrepeatable reads
		Защита от Phantom reads (при повторном чтении количество возвращающихся данных тот же)
