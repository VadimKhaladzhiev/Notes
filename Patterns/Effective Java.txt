Создание и удаление объектов

	1. Static фабричные методы вместо конструкторов
		Пример:
		public static Boolean valueOf(boolean b) {
			return b ? Boolean.TRUE : Boolean.FALSE;
		}
		
		Преимущества
		- Имеют имена
		- Не обязательно создают новые объекты
		- Могут вернуть экземпляры подклассов
		
		Недостатки
		- Если нет Public или Protected конструкторов 
		класс не может наследуемым
		- Статичные фабричные методы сложно визуально отделить от других 
		статичныз методов
		
	2. Builder вместо конструктора со множеством параметров
		Замена телескопических параметров и Java Beans pattern
		
	3. Замена поле синглтона private конструктором или enum type
